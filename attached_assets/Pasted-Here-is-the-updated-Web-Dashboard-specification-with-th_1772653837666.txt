Smart Home Dashboard — Web UI Specification

Overview

Design and implement a web-based dashboard UI for a smart home system. The goal is to create a clean, minimalist, and modern interface that provides clear insight into key system metrics and allows quick interaction with core components.

The dashboard should use a modular card-based layout consisting of four main cards. The architecture must allow easy expansion with additional cards in the future without major layout changes.

The interface should use clear typography, intuitive icons, and carefully chosen colors to present system information in a way that is both visually appealing and easy to understand.

The dashboard must fully support Light Mode and Dark Mode.

⸻

General Requirements

• Responsive web design optimized for mobile first, but scalable to tablets and desktop screens
• Minimalist and clean layout
• Full Light Mode and Dark Mode support
• Modular card-based architecture
• Easy to add additional cards in the future
• Smooth interactions and modern visual style
• Clear iconography for quick understanding
• Status colors used carefully:
	•	Normal
	•	Active
	•	Warning
	•	Error
	•	Inactive

⸻

Dashboard Layout

The dashboard contains four primary cards displayed in a responsive grid layout.

Layout behavior:

Mobile:
• Cards stacked vertically

Tablet / Desktop:
• 2×2 grid layout

⸻

Top Navigation / Header

The dashboard should include a minimal top header containing:

• Application title or logo
• Optional connection status indicator
• Logout button

Logout Button

Purpose: Allow the user to securely exit the dashboard.

UI suggestions:

• Icon + text (Logout)
• Positioned in the top-right corner
• Use a clear logout icon (arrow leaving door)
• Optional confirmation dialog

Example:

Logout → confirm dialog
“Are you sure you want to log out?”

1. Temperature Card

Purpose

Display environmental conditions from indoor and outdoor sensors.

⸻

Display

Indoor Sensor

• Temperature (°C)
• Humidity (%)
• Sensor battery level (%)

Outdoor Sensor

• Temperature (°C)
• Humidity (%)
• Sensor battery level (%)

⸻

UI Suggestions

• Thermometer icon for temperature
• Droplet icon for humidity
• Battery icon with percentage indicator
• Clearly separated Indoor and Outdoor sections

Optional color indications:

Temperature
• Blue → cold
• Neutral → comfortable
• Orange/Red → warm

Humidity
• Neutral color (blue/teal)

Battery
• Green → good
• Yellow → medium
• Red → low

⸻

Example Layout

Indoor

Temperature: 22.4 °C
Humidity: 45 %
Battery: 87 %

Outdoor

Temperature: 5.8 °C
Humidity: 71 %
Battery: 72 %

2. Diesel Heater Card

Purpose

Monitor and control the diesel heater.

⸻

Display

• Heater status
	•	On
	•	Off
	•	Error

• Current power level
• Core temperature (°C)
• Error message (if present)

⸻

Controls

• Power toggle button (On / Off)
• Increase power button (+)
• Decrease power button (–)

⸻

UI Suggestions

• Flame icon
• Highlight active heating state
• Red color for error states
• Visual power level indicator (bars or steps)

3. Battery Card

Purpose

Display battery and electrical system data.

⸻

Display

• State of Charge (SOC, %)
• Voltage (V)
• Current (A)
• Power (W)
• Battery Temperature 1 (°C)
• Battery Temperature 2 (°C)

⸻

UI Suggestions

• Battery icon
• SOC displayed as a progress bar
• Charging / discharging indicators

Example indicators:

Charging → green arrow
Discharging → orange arrow
Idle → neutral icon
